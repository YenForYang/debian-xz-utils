\subsubsection{L\+Z\+M\+A\+\_\+\+I\+G\+N\+O\+R\+E\+\_\+\+C\+H\+E\+CK}
{\footnotesize\ttfamily \#define L\+Z\+M\+A\+\_\+\+I\+G\+N\+O\+R\+E\+\_\+\+C\+H\+E\+CK~U\+I\+N\+T32\+\_\+C(0x10)}

This flag makes \doxyref{lzma\+\_\+code()}{p.}{base_8h_a28cc09bc422d5ba1e0187c9f2af5d957} not calculate and verify the integrity check of the compressed data in .xz files. This means that invalid integrity check values won\textquotesingle{}t be detected and L\+Z\+M\+A\+\_\+\+D\+A\+T\+A\+\_\+\+E\+R\+R\+OR won\textquotesingle{}t be returned in such cases.

This flag only affects the checks of the compressed data itself; the C\+R\+C32 values in the .xz headers will still be verified normally.

Don\textquotesingle{}t use this flag unless you know what you are doing. Possible reasons to use this flag\+:


\begin{DoxyItemize}
\item Trying to recover data from a corrupt .xz file.
\item Speeding up decompression, which matters mostly with S\+H\+A-\/256 or with files that have compressed extremely well. It\textquotesingle{}s recommended to not use this flag for this purpose unless the file integrity is verified externally in some other way.
\end{DoxyItemize}

Support for this flag was added in liblzma 5.\+1.\+4beta. \mbox{\label{container_8h_a563c84b5f368b3dd00d92ea903c5c33d}}

\subsubsection{L\+Z\+M\+A\+\_\+\+C\+O\+N\+C\+A\+T\+E\+N\+A\+T\+ED}
{\footnotesize\ttfamily \#define L\+Z\+M\+A\+\_\+\+C\+O\+N\+C\+A\+T\+E\+N\+A\+T\+ED~U\+I\+N\+T32\+\_\+C(0x08)}

This flag enables decoding of concatenated files with file formats that allow concatenating compressed files as is. From the formats currently supported by liblzma, only the .xz format allows concatenated files. Concatenated files are not allowed with the legacy .lzma format.

This flag also affects the usage of the `action\textquotesingle{} argument for \doxyref{lzma\+\_\+code()}{p.}{base_8h_a28cc09bc422d5ba1e0187c9f2af5d957}. When L\+Z\+M\+A\+\_\+\+C\+O\+N\+C\+A\+T\+E\+N\+A\+T\+ED is used, \doxyref{lzma\+\_\+code()}{p.}{base_8h_a28cc09bc422d5ba1e0187c9f2af5d957} won\textquotesingle{}t return L\+Z\+M\+A\+\_\+\+S\+T\+R\+E\+A\+M\+\_\+\+E\+ND unless L\+Z\+M\+A\+\_\+\+F\+I\+N\+I\+SH is used as `action\textquotesingle{}. Thus, the application has to set L\+Z\+M\+A\+\_\+\+F\+I\+N\+I\+SH in the same way as it does when encoding.

If L\+Z\+M\+A\+\_\+\+C\+O\+N\+C\+A\+T\+E\+N\+A\+T\+ED is not used, the decoders still accept L\+Z\+M\+A\+\_\+\+F\+I\+N\+I\+SH as `action\textquotesingle{} for \doxyref{lzma\+\_\+code()}{p.}{base_8h_a28cc09bc422d5ba1e0187c9f2af5d957}, but the usage of L\+Z\+M\+A\+\_\+\+F\+I\+N\+I\+SH isn\textquotesingle{}t required.
